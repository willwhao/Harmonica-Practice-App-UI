# P3 灰度发布与运营报表基础版方案

本方案覆盖 P3 “灰度发布、远程功能开关、运营报表和生产监控”的基础能力。当前阶段先建立服务端配置与指标汇总底座，不直接实现完整运营后台 UI。

## 已实现范围

- 新增 `release_flags` 表，保存远程功能开关、灰度比例、备注和更新时间。
- 新增公开读取接口 `GET /api/release-config`，前端可获取远程开关配置。
- 新增登录写入接口 `PUT /api/release-config/:flagKey`，用于更新指定功能开关。
- 前端 `buildReleaseConfig` 支持默认值、远程配置、环境变量覆盖和稳定灰度桶。
- 新增 `GET /api/ops/summary`，汇总用户、近 7 天活跃、练习、学习、CMS 内容和远程开关数量。
- 新增 `src/app/ops/opsClient.ts`，封装远程开关和运营摘要读取。

## 灰度规则

灰度使用稳定哈希桶：

1. 输入为 `flagKey + audienceKey`。
2. 计算 0–99 的 bucket。
3. 当 `bucket < rolloutPercent` 时命中灰度。

这意味着同一个用户在同一个功能上会稳定命中或不命中，不会因为刷新页面而随机变化。

优先级：

1. 通道默认值。
2. 服务端远程配置。
3. 环境变量覆盖。

环境变量仍保留最高优先级，方便生产紧急关闭某个能力。

## 当前限制

- `PUT /api/release-config/:flagKey` 当前只要求登录，后续需要补充管理员角色。
- 运营摘要是实时 SQL 聚合，适合内测和小规模公测；正式运营应转为定时聚合或数据仓库。
- 错误监控事件当前仍以接收和日志为主，尚未持久化到报表。
- 尚未实现后台 UI、告警通知和自动回滚。

## 后续升级建议

- 增加管理员权限与操作审计。
- 将监控事件持久化，按版本、通道、页面和浏览器聚合。
- 增加日活、留存、练习完成率、内容转化漏斗和授权音频播放指标。
- 将灰度策略接入 CI/CD 发布流程，支持自动扩大比例和一键回滚。
