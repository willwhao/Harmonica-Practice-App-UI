# 回滚计划

当生产版本出现白屏、登录不可用、练习流程不可用、数据同步异常或严重性能退化时，使用本计划。

## 触发条件

- 登录、注册、历史同步或练习入口大面积失败。
- 移动端首屏或练习页持续白屏。
- 新版本导致用户数据无法读取、导出或注销。
- 性能预算明显超标且影响实际使用。
- 版权、隐私或合规内容出现错误。

## 回滚步骤

1. 暂停继续发布，保留当前错误现场。
2. 在 GitHub Actions 中确认失败版本的提交 SHA、构建产物和日志。
3. 找到最近一个稳定 tag，例如 `v0.1.0`。
4. 将部署目标回退到稳定 tag 对应的构建产物。
5. 清理或刷新 CDN/静态资源缓存。
6. 验证以下路径：
   - 首页加载
   - 游客进入练习准备页
   - 登录/注册
   - 找回密码
   - 练习结果保存
   - 云端历史同步
7. 记录事故原因、影响范围、回滚时间和后续修复任务。

## 数据注意事项

- 不要直接删除生产数据库。
- 如果涉及数据库结构变更，必须先备份再执行迁移或回滚。
- 如果新版本写入了不兼容数据，需要单独编写数据修复脚本，并先在备份上演练。

## 恢复发布

完成修复后，重新执行：

```bash
npm run quality
npm run check:release
```

确认 CI 通过后，再创建新的修复版本 tag。
